# @bufbuild/protoc-gen-grpc-js

The code generator plugin for [@grpc/grpc-js](https://www.npmjs.com/package/@grpc/grpc-js), the official gRPC implementation for Node.js, with Protobuf messages generated by [protoc-gen-es](https://www.npmjs.com/package/@bufbuild/protoc-gen-es). Learn more about the project at [protobufes.com](https://protobufes.com/).

## Installation

`protoc-gen-grpc-js` generates gRPC service definitions, clients, and server types for every service in your Protocol Buffer schema. The generated code uses messages generated by [protoc-gen-es](https://www.npmjs.com/package/@bufbuild/protoc-gen-es), and requires the runtime libraries [@bufbuild/protobuf](https://www.npmjs.com/package/@bufbuild/protobuf) and [@grpc/grpc-js](https://www.npmjs.com/package/@grpc/grpc-js) version 1.9 or later. It's compatible with Protocol Buffer compilers like [buf](https://github.com/bufbuild/buf) and [protoc](https://github.com/protocolbuffers/protobuf/releases).

To install the runtime libraries and the plugins, run:

```shell
npm install @bufbuild/protobuf @grpc/grpc-js
npm install --save-dev @bufbuild/protoc-gen-es @bufbuild/protoc-gen-grpc-js
```

Since `@grpc/grpc-js` runs on Node.js, the generated code uses the `Buffer` global, and requires Node.js types to be available to the TypeScript compiler—for example, with `npm install --save-dev @types/node`.

## Generating code

### With buf

```bash
npm install --save-dev @bufbuild/buf
```

Add a new `buf.gen.yaml` configuration file:

```yaml
# Learn more: https://buf.build/docs/configuration/v2/buf-gen-yaml
version: v2
inputs:
  - directory: proto
plugins:
  # This will invoke protoc-gen-es and write output to src/gen
  - local: protoc-gen-es
    out: src/gen
    opt: target=ts
  # This will invoke protoc-gen-grpc-js and write output to src/gen
  - local: protoc-gen-grpc-js
    out: src/gen
    opt: target=ts
```

To generate code for all Protobuf files within your project, run:

```bash
npx buf generate
```

Note that `buf` can generate from various [inputs](https://buf.build/docs/reference/inputs),
not just local Protobuf files.

### With `protoc`

```bash
PATH=$PATH:$(pwd)/node_modules/.bin \
  protoc --proto_path proto \
  --es_out src/gen \
  --es_opt target=ts \
  --grpc-js_out src/gen \
  --grpc-js_opt target=ts \
  proto/a.proto proto/b.proto proto/c.proto
```

Note that `node_modules/.bin` needs to be added to the `$PATH` so that the Protobuf compiler can find the plugins. This happens automatically with npm scripts and with `npx`.

## Generated code

For every Protobuf file that contains services, the plugin generates a `_grpc.ts` file next to the `_pb.ts` file from `protoc-gen-es`. For example, the following service:

```protobuf
syntax = "proto3";
package example;

service ElizaService {
  rpc Say(SayRequest) returns (SayResponse);
}
```

generates the following exports in `eliza_grpc.ts`:

- `ElizaServiceDefinition`: The [@grpc/grpc-js](https://www.npmjs.com/package/@grpc/grpc-js) service definition, with serialization backed by [@bufbuild/protobuf](https://www.npmjs.com/package/@bufbuild/protobuf).
- `ElizaServiceServer`: An interface for the server implementation.
- `ElizaServiceClient`: A client with a typed method for every RPC.

To implement a server:

```ts
import * as grpc from "@grpc/grpc-js";
import { create } from "@bufbuild/protobuf";
import { SayResponseSchema } from "./gen/eliza_pb.js";
import { ElizaServiceDefinition, type ElizaServiceServer } from "./gen/eliza_grpc.js";

const implementation: ElizaServiceServer = {
  say(call, callback) {
    callback(null, create(SayResponseSchema, { sentence: `You said: ${call.request.sentence}` }));
  },
};

const server = new grpc.Server();
server.addService(ElizaServiceDefinition, implementation);
server.bindAsync("localhost:50051", grpc.ServerCredentials.createInsecure(), () => {});
```

To call it with a client:

```ts
import * as grpc from "@grpc/grpc-js";
import { create } from "@bufbuild/protobuf";
import { SayRequestSchema } from "./gen/eliza_pb.js";
import { ElizaServiceClient } from "./gen/eliza_grpc.js";

const client = new ElizaServiceClient("localhost:50051", grpc.credentials.createInsecure());
client.say(create(SayRequestSchema, { sentence: "Hi" }), (error, response) => {
  if (error) {
    throw error;
  }
  console.log(response?.sentence);
});
```

## Plugin options

`protoc-gen-grpc-js` supports the same options as `protoc-gen-es`: [`target`](https://github.com/bufbuild/protobuf-es/tree/main/packages/protoc-gen-es#target), [`import_extension`](https://github.com/bufbuild/protobuf-es/tree/main/packages/protoc-gen-es#import_extension), [`js_import_style`](https://github.com/bufbuild/protobuf-es/tree/main/packages/protoc-gen-es#js_import_style), [`keep_empty_files`](https://github.com/bufbuild/protobuf-es/tree/main/packages/protoc-gen-es#keep_empty_filestrue), [`ts_nocheck`](https://github.com/bufbuild/protobuf-es/tree/main/packages/protoc-gen-es#ts_nochecktrue), and [`elide_plugin_version`](https://github.com/bufbuild/protobuf-es/tree/main/packages/protoc-gen-es#elide_plugin_versiontrue).

Make sure to use the same `target` and `import_extension` values for both plugins, so that the generated `_grpc` files can import from the generated `_pb` files.
